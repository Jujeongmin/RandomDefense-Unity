# 메인화면 타이틀 리뉴얼 설계

날짜: 2026-07-14
대상: `Assets/Scenes/MainScene.unity` (720×1280 포트레이트)

## 문제

출시를 앞둔 메인화면에 세 가지 문제가 있다.

1. **배경이 휑하다.** `MainPage`의 `Mid` 영역(높이 769px)이 통째로 비어 있다.
2. **타이틀 로고가 없다.** 스토어 피처 그래픽에는 "운빨 랜덤 디펜스 : 운랜디" 로고가 있는데 게임 안에는 없다.
3. **버튼이 촌스럽다.** 단색 사각형에 코드로 색만 칠해져 있다.

부수적으로, `defaultScreenOrientation: 4`(자동회전, 가로 허용)인데 두 씬 모두 720×1280 세로 기준이라 기기를 눕히면 UI가 깨진다.

## 제약

- 사용자가 씬 UI를 직접 배치해 왔으므로 코드는 원칙적으로 배치를 건드리지 않는다. **이번 작업에 한해 메인화면 씬 UI 수정을 허가받았다.**
- 프로젝트는 빌트인 렌더 파이프라인이다. URP 셰이더로 변환하지 않는다.
- 새 아트를 그리지 않는다. 이미 있는 에셋만 쓴다.

## 가진 에셋

- `Assets/Store/ruby-portrait-1080x1920.png` — 다크 네이비/퍼플 배경의 루비 키아트. 9:16이라 캔버스와 정확히 일치.
- `Assets/GData/Image/Store/feature-graphic-1024x500-v2.png` — 타이틀 로고가 포함된 스토어 피처 그래픽.
- `Assets/Down/Universal Stylized UI/` — 광택 스타일 버튼 아틀라스(네이비 외곽선).

## 설계

### 배경
캔버스 루트에 풀스크린 `Background` 이미지를 페이지들보다 뒤에 배치하고 루비 키아트를 넣는다. 상점·연구 페이지에도 같은 배경이 깔린다. 그 위에 세로 그라디언트 오버레이(위·아래 어둡게, 가운데 투명)를 얹어 가독성을 확보한다.

루비는 화면 상단 기준 약 380~900px에 위치하는데, 이는 비어 있는 `Mid` 영역과 겹친다. 기존 UI를 하나도 옮기지 않고 배경만 깔아도 빈 공간이 주인공 아트로 채워진다.

### 타이틀 로고
피처 그래픽에서 로고를 투명 배경 PNG로 추출(휘도 기반 알파 키잉)해 `Assets/GData/Image/UI/title-logo.png`로 저장한다. `Mid` 상단(화면 위에서 130~330px)에 배치 — 루비와 겹치지 않는다.

### 버튼
Universal Stylized UI 스프라이트로 9-slice 교체.

| 버튼 | 스타일 |
| --- | --- |
| START | 골드/옐로우 — 가장 밝고 크게, 시선 유도 |
| MODE / ODDS / RANKING | 네이비 — 보조, 브랜드 외곽선색과 일치 |
| 설정 | 작은 정사각 |

ODDS·RANKING은 현재 `MainMenuManager`가 런타임에 생성한다. 이를 씬에 미리 배치하고 코드는 참조만 하도록 바꾼다.

### 모션
신규 `MainScreenPresenter.cs`: 로고 등장(위에서 떨어지며 페이드), START 버튼 은은한 펄스, 배경 아주 느린 호흡 스케일.

씬 배치값을 영구 변경하지 않는다. 원본 위치·스케일을 기억했다가 그 기준으로만 오프셋을 준다.

### 화면 방향
`defaultScreenOrientation`을 Portrait로 고정하고 가로 자동회전을 해제한다.

### 작업 방식
씬 YAML을 직접 수정하지 않는다. 프로젝트에 이미 있는 `Assets/Scripts/Editor/PreplacedUiBuilder.cs` 패턴대로 에디터 메뉴 커맨드를 만들어 실행한다. 결과가 마음에 들지 않으면 씬을 저장하지 않고 되돌릴 수 있다.

## 검토한 대안

- **라이브 디오라마** — 메인화면에 유닛 프리팹을 세우고 idle 애니메이션 + 배경 스크롤. 임팩트는 크지만 씬 구조를 크게 건드려 출시 직전에 부담이 크다. 생동감은 가벼운 모션으로 충분히 흉내낼 수 있다고 판단해 제외.
- **최소 손질** — 배경 이미지와 버튼 스프라이트만 교체. 빠르지만 로고 부재 문제가 남는다.
